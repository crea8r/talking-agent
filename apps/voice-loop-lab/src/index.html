App 2 · Voice Loop Lab

Probe the spoken turn loop before we spend tokens.

This is the non-LLM path: browser mic, browser speech recognition, deterministic replies, browser speech synthesis, and latency instrumentation. The goal is not intelligence. The goal is to learn whether the voice loop feels fast and controllable enough to deserve a later model.

This app uses browser-native speech APIs. It is best tested in Chrome. If speech recognition is unavailable, the typed input path still exercises the reply and TTS loop.

Support

This app is supposed to fail clearly when the browser cannot do speech recognition.

  • Chrome exposes the widest support for browser speech recognition.
  • Speech synthesis voices are device-specific, so voice quality varies.
  • No LLM is involved here, so every response is generated by deterministic rules.
  • If this loop feels too slow or too fragile, adding a model later will not save it.
