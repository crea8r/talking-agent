App 1 · LiveKit Self-Host Probe

Probe the room layer before we build the agent.

This replaces the public Jitsi experiment with a self-hosted LiveKit spike. The goal is narrower now: prove that a human browser client can mint a token, join a room, publish media, and see future machine participants on infrastructure we control.

This app is local-only. It binds to 127.0.0.1:4381 and can mint dev tokens through the local server. It is not production auth. The sample agent below is an in-page publisher for the prerecorded media, not the final headless runtime.

Connection Setup

Use a self-hosted LiveKit URL. For a local dev server, the default is ws://127.0.0.1:7880.
